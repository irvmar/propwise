## Slide 10: Technology & Architecture

**Built for reliability, security, and scale.**

### Key Talking Points:
- **AI Engine:** Anthropic Claude (Sonnet for real-time SMS responses -- optimal balance of speed, quality, and cost)
- **SMS Infrastructure:** Twilio Programmable Messaging -- enterprise-grade delivery with 99.95% uptime
- **Backend:** Firebase Cloud Functions (TypeScript) -- serverless, auto-scaling, zero infrastructure management
- **Database:** Firestore -- real-time sync, automatic scaling, strong consistency
- **Frontend:** Next.js 15 (App Router) -- fast, modern dashboard experience
- **Payments:** Stripe subscriptions with metered billing support
- **Architecture:** Monorepo with shared type contracts -- ensures consistency across frontend, backend, and AI layers

### Why This Stack Matters for Investors:
- **Low infrastructure costs:** Serverless = pay per use, not per server. Scales linearly with revenue.
- **Fast iteration speed:** Monorepo + TypeScript = ship features quickly with confidence
- **AI vendor flexibility:** Agent architecture allows swapping LLM providers if needed (not locked into one vendor)
- **Security:** Firebase Auth + Firestore security rules + E.164 phone number validation. SOC 2 compliance roadmap in place.

Clean architecture diagram showing: Tenant SMS <-> Twilio <-> Firebase Functions <-> Claude AI <-> Firestore <-> Next.js Dashboard <-> Property Manager.
